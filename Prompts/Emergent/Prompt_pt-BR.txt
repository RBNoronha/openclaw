<CONFIGURAÇÃO DE AMBIENTE>
1. Arquitetura de Serviço e Configuração de URLs:
    - Este é um app full-stack com frontend React, backend FastAPI e banco MongoDB
    - VARIÁVEIS DE AMBIENTE PROTEGIDAS (NÃO MODIFICAR):
        • frontend/.env: REACT_APP_BACKEND_URL (URL externa configurada para produção)
        • backend/.env: MONGO_URL (configurado para acesso local ao MongoDB)
    - REGRAS DE USO DE URL:
        1. Banco de dados: DEVE USAR APENAS MONGO_URL existente em backend/.env
        2. Chamadas de API do frontend: DEVE USAR APENAS REACT_APP_BACKEND_URL
        3. Binding do backend: DEVE permanecer em 0.0.0.0:8001 (supervisor gerencia mapeamento externo)
        4. NUNCA modifique URLs ou portas em arquivos .env
        5. NUNCA codifique URLs ou portas no código
        6. Todas as rotas de API do backend DEVEM ser prefixadas com '/api' para coincidir com regras de ingress do Kubernetes que redirecionam essas requisições para a porta 8001

    - CONFIGURAÇÃO DO SERVIÇO:
        • Backend roda internamente em 0.0.0.0:8001 via supervisor
        • Esta porta interna está corretamente mapeada para REACT_APP_BACKEND_URL
        • Frontend acessa backend SOMENTE via REACT_APP_BACKEND_URL
        • Backend acessa MongoDB SOMENTE via MONGO_URL

    - USO DE VARIÁVEIS DE AMBIENTE:
        • Frontend: import.meta.env.REACT_APP_BACKEND_URL ou process.env.REACT_APP_BACKEND_URL
        • Backend: os.environ.get('MONGO_URL')

    - Controle do serviço:
        • sudo supervisorctl restart frontend/backend/all

    - IMPORTANTE: Comportamento de Hot Reload:
       - Frontend e backend têm hot reload habilitado
       - Reinicie servidores somente quando:
            * Instalando novas dependências ou salvando algo em .env

    - Regras de Ingress do Kubernetes:
        1. Todas as rotas de API do backend são redirecionadas para a porta 8001 quando prefixadas com '/api'
        2. Rotas do frontend (sem prefixo '/api') são direcionadas para a porta 3000
        3. Não usar o prefixo '/api' resultará em roteamento incorreto e falhas de serviço

Nota importante sobre URLs e .env:
- A URL do backend está armazenada em .env como REACT_APP_BACKEND_URL no diretório frontend. Use isso para todos os casos. Não codifique a URL do backend no código.
